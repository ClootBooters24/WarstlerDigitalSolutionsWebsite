Accessibility Statement

Our commitment to web accessibility for everyone.

Last Updated: February 5, 2026

Our Commitment

Warstler Digital Solutions is committed to ensuring that our website and services are accessible to all people, including those with disabilities. We believe that accessibility is not only a legal requirement but a moral imperative to create an inclusive digital environment.

Accessibility Standards

We strive to conform to the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA published by the World Wide Web Consortium (W3C). WCAG 2.1 Level AA is the globally recognized standard for web accessibility and is aligned with the Americans with Disabilities Act (ADA) Title III requirements in the United States.

Accessibility Features

Our website includes the following accessibility features:

Visual Design

  • Color Contrast: Text and interactive elements meet WCAG AA standards for color contrast (4.5:1 for normal text)
  • Readable Fonts: Clear, sans-serif fonts with sufficient font sizes for readability
  • Resizable Text: Visitors can resize text in their browser without loss of functionality
  • Visual Indicators: Links are underlined or otherwise visually distinct, not relying solely on color

Navigation and Structure

  • Logical Structure: Pages use proper heading hierarchies (H1, H2, H3, etc.) for organization
  • Keyboard Navigation: All pages are fully navigable using keyboard only (Tab, Enter, Arrow keys)
  • Skip Links: Skip navigation links allow users to bypass repetitive content
  • Clear Focus Indicators: Interactive elements show clear focus states when navigated by keyboard

Media and Images

  • Alternative Text: All images include descriptive alt text for screen readers
  • Videos: Embedded videos include captions and transcripts when applicable
  • Complex Graphics: Charts and diagrams include detailed text descriptions

Forms and Input

  • Associated Labels: All form fields have properly associated labels
  • Clear Instructions: Form requirements and errors are clearly communicated
  • Error Identification: Form validation errors are identified and described in text
  • Error Correction: Users can correct errors and resubmit

Screen Reader Compatibility

  • All pages are compatible with major screen readers (NVDA, JAWS, VoiceOver)
  • Semantic HTML is used throughout for proper content interpretation
  • Hidden text provides context not visible on screen

Known Limitations

While we strive for full accessibility, some limitations may exist:

  • User-Generated Content: Content provided by clients may not always meet accessibility standards. We work with clients to improve content accessibility.
  • Third-Party Plugins: Some third-party tools and plugins may have accessibility limitations beyond our control.
  • Legacy Features: Older website features may not fully meet current accessibility standards, though we continuously work to improve them.

Accessibility in Our Services

When designing and developing websites for clients, we:

  • Build all sites to WCAG 2.1 Level AA standards as a baseline
  • Include accessibility training and documentation for clients
  • Provide recommendations for accessible content authoring
  • Offer ongoing accessibility audits and improvements
  • Stay current with accessibility best practices and standards

Assistive Technology Support

Our website has been tested for compatibility with the following assistive technologies:

  • Screen Readers: NVDA, JAWS, VoiceOver, TalkBack
  • Browser Extensions: Color contrast enhancers, text-to-speech tools
  • Voice Control: Voice navigation compatible
  • Zoom Software: Compatible with text and page zoom functions

Browser and Device Support

Our website is accessible across:

  • Recent versions of major browsers (Chrome, Firefox, Safari, Edge)
  • Mobile devices (iOS, Android) with built-in accessibility features
  • Operating system accessibility tools (Windows High Contrast, Mac accessibility features)

Testing and Evaluation

We regularly conduct accessibility testing including:

  • Automated accessibility scans using industry tools
  • Manual testing with assistive technologies
  • Keyboard-only navigation testing
  • Screen reader compatibility testing
  • User feedback from people with disabilities

Continuous Improvement

Accessibility is an ongoing commitment. We continuously monitor our website for accessibility issues and regularly update our practices based on:

  • User feedback and requests
  • Updates to WCAG standards
  • Emerging accessibility best practices
  • Changes in assistive technology capabilities

Reporting Accessibility Issues

If you encounter any accessibility barriers while using our website or services, please report them so we can address them promptly. We appreciate your feedback and will work to resolve issues as quickly as possible.

How to Report an Issue:

  • Email: cwarstler@warstlerdigital.com (Subject: "Accessibility Issue")
  • Phone: 620-778-5786
  • Contact Form: Use our contact page to report an issue

When Reporting, Please Include:

  • The specific page or feature that is inaccessible
  • What assistive technology you are using (if applicable)
  • A description of the barrier you encountered
  • Your preferred method of contact

Response to Accessibility Issues

Upon receiving an accessibility issue report, we will:

  • Acknowledge receipt within 2 business days
  • Investigate the reported issue
  • Work toward a resolution or provide an explanation
  • Keep you informed of our progress
  • Implement a fix within a reasonable timeframe (typically 30 days for non-critical issues)

Legal References

This accessibility statement and our efforts are aligned with:

  • Americans with Disabilities Act (ADA): Title III (public accommodations)
  • Section 508 Rehabilitation Act: Electronic and information technology accessibility
  • Web Content Accessibility Guidelines (WCAG) 2.1: W3C standard

Limitations of Technical Accessibility Solutions

While we employ technical solutions to enhance accessibility, we recognize that:

  • AI and Plugins: Automated accessibility enhancement tools have limitations and may not address all accessibility needs
  • Manual Effort Required: True accessibility requires manual testing and refinement by accessibility professionals
  • User Diversity: Accessibility needs vary greatly among individuals with disabilities
  • Ongoing Process: Accessibility is not a destination but a continuous practice
